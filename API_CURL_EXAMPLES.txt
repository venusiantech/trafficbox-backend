================================================================================
STATUS VALUES REFERENCE
================================================================================

CUSTOM PLAN REQUEST STATUSES:
- "pending"   - User submitted, waiting for review
- "reviewed"  - Admin has seen it
- "approved"  - Approved, custom plan being created
- "rejected"  - Not approved

CONTACT US MESSAGE STATUSES:
- "new"       - Just received
- "read"      - Admin has read it
- "replied"   - Admin responded to user
- "archived"  - Closed/archived

NOTIFICATION TYPES:
- "custom_plan_request_submitted"  - User submitted custom plan request
- "custom_plan_request_updated"    - Admin updated request status
- "contact_us_submitted"           - User submitted contact message
- "contact_us_replied"             - Admin replied to message

================================================================================
TIPS & NOTES
================================================================================

1. TOKEN USAGE:
   - Get JWT token by logging in first
   - Include token in Authorization header: "Bearer YOUR_JWT_TOKEN"
   - Admin endpoints require admin role

2. OPTIONAL AUTHENTICATION:
   - Custom plan request works with OR without JWT
   - Contact us works with OR without JWT
   - If email matches existing user, automatically linked

3. AUTOMATIC NOTIFICATIONS:
   - Created when user submits form
   - Created when admin updates status (approved, rejected, replied)
   - User can view, mark as read, or delete

4. PAGINATION:
   - Default: page=1, limit=20
   - Max limit: 20 items per page
   - Use ?page=2&limit=10 for custom pagination
